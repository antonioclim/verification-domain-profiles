# Limitations

1. Complete-domain optimisation is faster on several small cases, so no universal speed advantage is claimed.
2. The large enumerated full-domain timing baseline uses floating-point HiGHS and is not a mathematical proof source.
3. The structured adapters cover assignment and layered acyclic paths rather than every combinatorial model.
4. Correctness cases cover two to five objectives; no higher-dimensional empirical extrapolation is made.
5. Finite termination is proved but no polynomial oracle-call bound is claimed.
6. The profile is invariant under consistent changes of units but depends on the declared scale policy. Scales are model data and must be fixed before domain comparisons.
7. Independent physical-machine replication is not claimed by this archive.
8. Hosted cross-platform continuous integration is configured and becomes externally observable only after the repository update.
9. A targeted literature search found no exact collision for the combined profile, dual and trichotomy construction, but exhaustive priority certification is not claimed.
10. Output-sensitive enumeration of maximal certifiable domains is outside the present contribution.
